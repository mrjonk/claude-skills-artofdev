# Exemple — Page produit

## Contexte

Page produit pour un SaaS générique fictif.
Sujet : *"Application de time tracking pour freelances"*.
Public : freelances 25–45 ans, secteur tech / créatif.

## Prompt utilisateur

> *"Create a product page for a fictional time-tracking SaaS for freelancers. Tone : product page (real benefits, concrete, credible, no vague promises). Avoid : 'the best', 'the ultimate', 'revolutionize'. Include H1, problem solved, concrete benefits, how it works in 3 steps, use cases, social proof placeholders, features table, pricing 3 plans, FAQ 7+ objections, internal links, image plan, 1 main CTA + 1 secondary, quality score."*

## Angle éditorial attendu

- **Bénéfices mesurables** : pas *"gagnez du temps"*, mais *"facturez en moyenne 4 h supplémentaires par semaine en éliminant la saisie manuelle"* ;
- **Concret** : flow montré en 3 étapes courtes, captures réelles ;
- **Crédible** : prix clairs, FAQ qui traite les vraies objections, testimonials placeholder seulement si réels.

## Structure attendue

| Section | Contenu |
|---|---|
| Hero | nom + bénéfice principal + 2 CTA |
| H2 — Le problème résolu | 2–3 phrases spécifiques |
| H2 — Bénéfices concrets | 3–4 bénéfices mesurables |
| H2 — Comment ça marche | 3 étapes avec captures |
| H2 — Cas d'usage | 2–3 personas |
| H2 — Preuves sociales | métriques, logos, testimonials (vrais uniquement) |
| H2 — Fonctionnalités | tableau factuel |
| H2 — Tarifs | 3 plans avec prix clairs |
| H2 — FAQ | 7+ objections traitées |
| H2 — Pour aller plus loin | liens internes |
| CTA final | action concrète |

## SEO attendu

- **title** : *"App de time tracking pour freelances — facturez ce que vous travaillez"* (66 car., **à raccourcir** un peu) ;
- **meta description** : *"App de suivi du temps conçue pour freelances. Démarrage automatique, factures clients en 1 clic, intégration Stripe. Essai gratuit."* (155 car.) ;
- **slug** : `time-tracking-freelance` ;
- **mot-clé principal** : *"time tracking freelance"* ;
- **search_intent** : transactionnelle ;
- **schema_type** : `Product` + `Offer`.

## Exemples concrets à inclure

- **Bénéfices chiffrés** (avec marquage TO VERIFY si non sourcés) ;
- **Captures réelles** de l'interface (3 captures minimum : démarrage timer, facturation, dashboard) ;
- **Cas d'usage** avec persona nommé (*"Marie, designer freelance, gère 3 clients en parallèle..."*) ;
- **Pricing transparent** : prix HT/TTC clairs, devise affichée, mention essai gratuit / engagement.

## Liens internes possibles

| # | Slug cible | Ancre | Raison |
|---|---|---|---|
| 1 | `/docs/getting-started` | "guide de démarrage rapide" | conversion → activation |
| 2 | `/integrations/stripe` | "intégration Stripe" | feature clé pour freelances |
| 3 | `/case-studies/freelance-designer` | "étude de cas designer" | preuve sociale |
| 4 | `/pricing` | "détail des tarifs" | si CTA tarif |

## Images recommandées

| Slot | Description | Style |
|---|---|---|
| Hero | Mockup app (timer en cours sur dashboard) | screenshot ou 3D mockup |
| Section "Comment ça marche" | 3 captures réelles des 3 étapes | screenshots |
| Section "Cas d'usage" | Photos sobres des personas (ou avatars génériques propres) | photo |
| Avatars testimonials | Photos clients **réels uniquement** (autorisation écrite) | portrait sobre |

## Sources nécessaires

> Une page produit cite **moins** de sources qu'un article de blog, mais doit être **factuelle** :

- statistiques de gain de temps : sourcer ou retirer ;
- mentions de conformité (RGPD, sécurité) : préciser les certifications réelles ;
- citations clients : avec autorisation écrite uniquement.

## Erreurs à éviter

- *"La solution ultime pour les freelances !"* (superlatif vide) ;
- *"Révolutionnez votre facturation !"* (verbe interdit) ;
- **bullets vagues** : *"Rapide. Simple. Puissant."* (à remplacer par des bénéfices mesurables) ;
- **testimonials inventés** ("Marie, freelance, 32 ans") sans existence réelle ;
- **pricing vague** : *"À partir de seulement..."* (sans donner le prix exact) ;
- **5 CTA différents** dispersés (1 principal + 1 secondaire suffit) ;
- **FAQ qui élude** les objections : *"Comment annuler ?"* qui répond *"Contactez-nous"* (pas acceptable, doit être self-service).

## Score qualité attendu

**75–88 / 100** — Bon contenu SEO. Premium si :
- testimonials **vrais** disponibles ;
- captures **réelles** de l'app ;
- chiffres de bénéfices **sourcés** (étude utilisateur, benchmark interne).
