# Exemple — Article SaaS

## Contexte

Article de blog SaaS pour fondateurs et product managers.
Sujet générique réutilisable : *"Pourquoi les petites équipes échouent avec les outils de suivi de projet (et comment éviter ça)"*.

## Prompt utilisateur

> *"Write a SaaS-style blog article about why small teams struggle with project tracking tools. Audience : product managers and small SaaS founders. Tone : SaaS, problem/solution, non-aggressive. Length : 1500 words. Structure : pose the problem with data, present an approach (not a product pitch), 2-3 anonymized case studies, common mistakes, FAQ, CTA, internal links, image plan, sources, quality score."*

## Angle éditorial attendu

- **Angle problème** : pourquoi 80 % des outils de gestion de projet sont abandonnés en 2 mois dans les petites équipes ;
- **Approche** : présenter une méthode pour évaluer un outil avant de l'adopter ;
- **Pas** un pitch produit : l'article doit être **utile** même sans le produit du blog hôte.

## Structure attendue

| Section | Contenu |
|---|---|
| H1 | angle problème ou bénéfice (pas un nom d'outil) |
| Introduction | constat + question implicite |
| H2 — Le problème en chiffres | données / études citées |
| H2 — Pourquoi les outils standards échouent | 3 raisons documentées |
| H2 — L'approche : comment évaluer un outil avant adoption | grille critères |
| H2 — Cas d'usage / mini-études (2–3) | situations types anonymisées |
| H2 — Erreurs à éviter | 4–5 erreurs |
| H2 — FAQ | 5–6 questions stratégiques |
| H2 — Pour aller plus loin | liens internes |
| Conclusion + CTA | invitation à la lecture / démo / guide complémentaire |

## SEO attendu

- **title** : *"Pourquoi les petites équipes abandonnent leurs outils de gestion"* (62 car.) ;
- **meta description** : *"4 raisons pour lesquelles les outils de project tracking échouent dans les petites équipes — et une grille pour évaluer un outil avant de l'adopter."* (160 car.) ;
- **slug** : `petites-equipes-outils-gestion` ;
- **mot-clé principal** : *"outils gestion projet petites équipes"* ;
- **search_intent** : problème à résoudre.

## Exemples concrets à inclure

- **Au moins 1 chiffre** sourcé ou marqué TO VERIFY (taux d'abandon, durée moyenne d'usage) ;
- **2 cas d'usage** anonymisés et **étiquetés comme tels** ("Voici une situation type observée régulièrement") ;
- **Tableau** comparatif si pertinent (critères d'évaluation).

## Liens internes possibles

| # | Slug cible | Ancre | Raison |
|---|---|---|---|
| 1 | `/blog/onboarding-equipe-tools` | "onboarder son équipe sur un nouvel outil" | suite logique après adoption |
| 2 | `/blog/grille-evaluation-saas` | "grille d'évaluation SaaS" | approfondit la méthodologie |
| 3 | `/case-studies` | "études de cas clients" | preuve sociale |
| 4 | `/demo` | "tester la démo" | CTA conversion |

## Images recommandées

| Slot | Description | Style |
|---|---|---|
| Hero | Schéma simple du problème (équipe paralysée par 5 outils) ou photo réaliste de réunion | illustration plate ou photo |
| Section "Cas d'usage" | Schéma avant/après ou tableau visuel | vectoriel |
| Section "Grille d'évaluation" | Tableau formaté ou check-list visuelle | vectoriel |

## Sources nécessaires

- Étude ou rapport sur l'abandon des outils SaaS (à citer ou marquer TO VERIFY) ;
- Article ou benchmark sectoriel sur la gestion de projet (à citer) ;
- Méthode d'évaluation établie (référencer un framework existant si applicable).

## Erreurs à éviter

- **introduction commençant par** *"Dans le monde du travail moderne..."* ;
- **affirmation sans source** : *"80 % des outils sont abandonnés"* sans nom d'étude ;
- **pitch produit déguisé** : *"Heureusement, notre solution résout tous ces problèmes !"* ;
- **CTA agressif** : *"Inscrivez-vous maintenant !"* ;
- **cas d'usage inventés** présentés comme réels (anonymiser **ou** étiqueter "situation type") ;
- **paragraphes transposables** (généralités sur la productivité valables pour n'importe quel sujet) ;
- **FAQ artificielle** avec des questions évidentes auxquelles l'article a déjà répondu.

## Score qualité attendu

**75–85 / 100** — Publiable avec corrections. Premium si l'angle est vraiment distinctif et les sources solides.
