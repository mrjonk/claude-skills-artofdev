# Guide d'écriture humaine — `multilingual-site-engine`

Comment écrire et réécrire des textes UI / web qui ne sentent pas l'IA.

Created by João de Almeida

---

## 1. Pourquoi humaniser

Les textes générés par IA partagent des tics reconnaissables :

- tirets cadratins (—) en boucle ;
- formules d'introduction passe-partout ;
- vocabulaire marketing creux ;
- structures rhétoriques répétitives ;
- conclusions qui résument sans apporter ;
- ton "neutre-fade" qui ne ressemble à personne.

Les utilisateurs reconnaissent ce style. Un site qui sent l'IA paraît **paresseux**, **non investi**, **interchangeable** — l'inverse de ce qu'on cherche pour une marque.

Le rôle du skill : repérer ces tics et les remplacer par une **voix humaine**.

## 2. Principes d'une bonne écriture humaine

### 2.1 Concrétude

Préférer le **concret** au générique :

| Générique | Concret |
|---|---|
| *"Améliorez votre productivité"* | *"Notez vos idées en 2 secondes, retrouvez-les en 5"* |
| *"Solution innovante"* | *"Un éditeur Markdown qui sauvegarde tout seul"* |
| *"Optimisez votre workflow"* | *"Plus de copier-coller entre 3 outils"* |
| *"Expérience fluide"* | *"L'app charge en moins d'une seconde"* |

### 2.2 Simplicité

- phrases **courtes** (15-20 mots max en moyenne) ;
- 1 idée par phrase ;
- vocabulaire **courant** (pas *"paradigme"*, *"écosystème"*, *"holistique"* sauf justifié).

### 2.3 Voix active

- *"Nous protégeons vos données"* > *"Vos données sont protégées par nos systèmes"* ;
- *"Tu peux annuler à tout moment"* > *"L'annulation est possible à tout moment"*.

### 2.4 Spécificité

- chiffres **précis** (*"1 200 utilisateurs actifs"* > *"des milliers d'utilisateurs"*) ;
- noms **propres** quand pertinent (*"Compatible avec Notion et Obsidian"* > *"Compatible avec les outils populaires"*) ;
- exemples **réels** (*"Par exemple, ajouter une dépense en 3 secondes"* > *"Par exemple, gérer ses finances facilement"*).

### 2.5 Naturel

- contractions si la marque le permet (*"On"* en FR informel, *"can't"* en EN) ;
- expressions du quotidien plutôt que jargon ;
- ponctuation locale **normale** (virgule, point — pas tiret long systématique).

## 3. Tics à supprimer

### 3.1 Le tiret cadratin abusif

**Signe distinctif** d'un texte généré par IA. À détecter :

- plus de **2 tirets longs** dans un même paragraphe ;
- tirets longs **systématiques** dans plusieurs paragraphes ;
- tiret long utilisé à la place d'une virgule ou d'un deux-points sans raison.

**Exemple à corriger :**

> *"Notre solution — pensée pour les équipes modernes — vous permet — en quelques clics — d'optimiser votre workflow."*

**Réécriture :**

> *"Notre app permet aux équipes de gagner du temps en quelques clics."*

**Règle :** le tiret long est utile pour une **incise rhétorique forte**, pas comme remplacement systématique de la ponctuation normale. En français, la virgule et les deux-points couvrent 95 % des cas.

### 3.2 Phrases creuses (FR)

| Phrase à bannir | Pourquoi |
|---|---|
| *"Dans le monde d'aujourd'hui"* | générique, sans valeur |
| *"À l'ère du numérique"* | cliché |
| *"Il est important de"* | mou, pédant |
| *"De plus en plus"* | flou, non chiffré |
| *"Découvrez comment"* | introduction de remplissage |
| *"Notre solution innovante"* | aucun contenu |
| *"Révolutionner"* | superlatif vide |
| *"Expérience utilisateur exceptionnelle"* | tautologie marketing |
| *"Optimiser votre productivité"* | passe-partout |
| *"Bienvenue sur notre site"* | inutile |
| *"En conclusion"* | écriture scolaire |

### 3.3 Phrases creuses (EN)

| Phrase à bannir | Pourquoi |
|---|---|
| *"In today's world"* | générique |
| *"In the digital age"* | cliché |
| *"It's important to"* | mou |
| *"More and more"* | flou |
| *"Discover how"* | remplissage |
| *"Innovative solution"* | vide |
| *"Revolutionize"* | superlatif |
| *"Seamless experience"* | tautologie |
| *"Boost your productivity"* | passe-partout |
| *"Welcome to our website"* | inutile |
| *"In conclusion"* | écriture scolaire |

### 3.4 Structures rhétoriques répétées

- *"Non seulement X, mais aussi Y"* à toutes les sauces ;
- *"Que ce soit pour A, B ou C..."* à répétition ;
- *"X. Y. Z. Voilà ce que nous proposons."* (énumération + chute) ;
- *"Imaginez un monde où..."* (intro fictionnelle creuse).

### 3.5 Conclusions vides

- résumés qui répètent l'intro ;
- *"En conclusion, X est essentiel."* ;
- *"Voilà tout ce qu'il faut savoir sur..."* ;
- CTA générique non lié au contenu.

## 4. Méthode de réécriture

### Étape 1 — Lire à voix haute

Si une phrase ne sonne pas naturelle à l'oral, elle ne sonnera pas naturelle à l'écrit.

### Étape 2 — Compter les tirets longs

Si plus de 2 par paragraphe sans raison, remplacer.

### Étape 3 — Test du paragraphe transposable

Si le paragraphe peut être copié sur un autre site, dans un autre secteur, sans rien changer → **trop générique**.

### Étape 4 — Couper

Supprimer tout mot qui n'apporte rien :

| Avant | Après |
|---|---|
| *"Il est tout à fait possible que..."* | *"Peut-être que..."* |
| *"Dans le but de"* | *"Pour"* |
| *"Au niveau de"* | *"Pour"* / *"Sur"* |
| *"Mettre en place"* | *"Faire"* / *"Installer"* |
| *"Procéder à la création"* | *"Créer"* |

### Étape 5 — Concrétiser

Remplacer chaque adjectif marketing par un **fait** :

- *"Rapide"* → *"Charge en 0,4 seconde"*
- *"Sécurisé"* → *"Chiffrement AES-256"*
- *"Facile à utiliser"* → *"Aucune installation, ouvre dans le navigateur"*

### Étape 6 — Garder une voix

Le site a-t-il une voix de marque ?

- **chaleureuse** (Notion, Linear) ;
- **professionnelle** (Stripe, AWS) ;
- **ludique** (Mailchimp, Slack) ;
- **précise** (Apple, Linear) ;
- **militante** (Patagonia, Lemonade) ?

Choisir et **garder cette voix** dans tout le texte.

## 5. Exemples avant / après

### Exemple 1 — Page produit SaaS

**Avant (IA) :**

> *"Dans le monde d'aujourd'hui — où chaque seconde compte — il est important de pouvoir compter sur des outils performants. Notre solution innovante — pensée pour les équipes modernes — révolutionne la manière dont vous gérez votre productivité. Découvrez comment optimiser votre workflow grâce à une expérience fluide et intuitive."*

**Après (humain) :**

> *"Notre app remplace 4 outils par un seul : notes, tâches, calendrier, doc partagé. Les équipes l'utilisent pour arrêter de jongler entre Notion, Asana, Google Calendar et Slack. 10 jours d'essai gratuit, pas de carte bancaire."*

Différence : 100 % concret, 0 mot creux, voix claire.

### Exemple 2 — Erreur de formulaire

**Avant (IA) :**

> *"Une erreur s'est produite. Veuillez réessayer ultérieurement ou contacter notre équipe support."*

**Après (humain) :**

> *"Mot de passe trop court : minimum 8 caractères."*

(L'erreur précise est plus utile que le message générique.)

### Exemple 3 — Email transactionnel

**Avant (IA) :**

> *"Cher utilisateur, nous avons le plaisir de vous informer que votre compte a été créé avec succès. Nous vous invitons à découvrir l'ensemble des fonctionnalités que notre plateforme met à votre disposition."*

**Après (humain) :**

> *"Bonjour João, ton compte est prêt. Pour démarrer : connecte ton calendrier en 30 secondes."*

## 6. Spécificités par langue

### Français
- pas de tiret long systématique (anglo-saxon) ;
- espaces insécables avant `: ; ! ?` ;
- guillemets français `« »` plutôt que `" "` (selon contexte) ;
- éviter les anglicismes inutiles (*"checkez"*, *"liker"*, *"booker"*).

### Anglais
- contractions normales (`don't`, `can't`, `we'll`) ;
- direct, peu d'incises ;
- éviter les superlatifs (*"the best"*, *"unique"*, *"world-class"*) sans preuve.

### Allemand
- précision et structure ;
- composés longs acceptés ;
- vouvoiement (Sie) par défaut sauf marque jeune.

### Espagnol
- tutoiement (tú) ou vouvoiement (usted) selon pays (LatAm vs Espagne) ;
- formel selon secteur ;
- ne pas mélanger les variantes régionales.

### Portugais
- pt-BR vs pt-PT : variantes distinctes (préférer une seule par site).

## 7. Quand garder un tiret long

Le tiret long **a sa place** quand :

- il marque une **rupture rhétorique forte** ;
- il introduit une **chute** ;
- il est utilisé une fois par texte, pas en boucle.

Exemple acceptable :

> *"Tout fonctionne dès le premier jour — jusqu'à ce qu'on ait besoin de personnaliser une vue."*

Le tiret long marque ici une vraie inflexion, pas un remplissage.

## 8. Checklist humanisation

- [ ] tirets longs comptés (≤ 1 par paragraphe sauf exception) ;
- [ ] phrases creuses supprimées ;
- [ ] adjectifs marketing remplacés par des faits ;
- [ ] introductions raccourcies ;
- [ ] conclusions vides supprimées ;
- [ ] voix de marque cohérente ;
- [ ] paragraphes transposables réécrits ;
- [ ] lecture à voix haute fluide ;
- [ ] longueur vs source raisonnable.
